## 基本工作流

LSPilot 的使用流程非常简单，遵循"提问 → 分析 → 生成 → 注入"的模式。

### 1. 提出需求

在 LSPilot 中用自然语言描述你想做的事情，例如：

- "帮我 Hook 掉这个应用的会员校验"
- "追踪这个构造函数的调用链"
- "查找处理登录请求的类和方法"

### 2. AI 自动分析

AI 会解析你的意图，并通过内置工具自动：

- 定位目标类和方法
- 分析代码逻辑
- 生成 Hook 脚本

### 3. 生成脚本

AI 会为你生成可直接运行的示例代码，并提供清晰的逆向思路说明。

### 4. 动态注入

通过 LSPosed 框架一键动态注入脚本，实时验证结果，无需反复编译。

## 插件脚本

插件存储在以下目录：

```
Android/media/<包名>/LSPilot/Plugin/
```

支持三种脚本语言：

- **BeanShell (Java)**：兼容 Java 语法，可直接调用 Android API
- **Lua**：轻量级 LuaJIT 引擎，适合快速编写调试脚本
- **JavaScript**：Rhino 引擎，支持现代 JS 语法

创建插件后可直接在应用内编辑运行。

## 使用技巧

### 描述需求要清晰

越清晰的描述，AI 分析越准确。尽量说明：

- 目标功能（会员校验、登录、支付等）
- 期望行为（绕过、修改、追踪等）
- 已知信息（类名、方法名、字符串等）

### 善用逆向分析工具

结合内置的 jadx 反编译、DexKit 搜索等工具，先获取必要信息再让 AI 生成脚本。

### 迭代调试

如果首次生成的脚本效果不理想，可以基于运行结果反馈给 AI 进行迭代优化。
